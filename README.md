# os-docs-v2

Шаблон документации на базе SvelteKit, MDsveX и Tailwind CSS.

Создавайте брендированные сайты документации с автогенерируемой навигацией, поиском, подсветкой синтаксиса, тёмной темой и полной SEO-поддержкой.

---

## Быстрый старт

```bash
npm install
npm run dev
```

Сервер разработки запускается автоматически. Откройте локальный URL из терминала. Главная страница редиректит на `/docs`.

## Что менять в первую очередь

Файлы, которые нужно обновить сразу после клонирования шаблона:

| Приоритет | Файл | Что менять |
|-----------|------|------------|
| 1 | `apps/web/src/lib/config/site.ts` | Имя сайта, URL, описание, автор, ссылки GitHub/email, ключевые слова |
| 2 | `apps/web/src/lib/config/branding.ts` | Имя бренда и пути к логотипам |
| 3 | `apps/web/src/routes/layout.css` | Акцентный цвет (`--accent` в `:root` и `.dark`), базовый оттенок, радиус |
| 4 | `apps/web/static/` | Замените `favicon.png`, `light_logo.png`, `dark_logo.png`, `og-image.jpg` на свои |
| 5 | `apps/web/static/site.webmanifest` | Имя приложения, описание, цвет темы |
| 6 | `apps/web/src/app.html` | `<html lang>`, дефолтный `<title>` |
| 7 | `package.json` / `apps/web/package.json` | Имя пакета, версия, описание |
| 8 | `README.md` | Этот файл — замените на README своего проекта |

## Где писать статьи

Весь контент документации находится в:

```
apps/web/src/lib/content/[N]Документация{docs}/
```

Директория использует систему имён с тремя уровнями:

| Префикс | Назначение | Пример |
|---------|------------|--------|
| `[N]` | **Секция** (верхний уровень) | `[N]Документация{docs}` → URL: `/docs` |
| `[C]` | **Категория** (вложенная группа) | `[C]Настройки{settings}` → группирует страницы под заголовком в боковой панели |
| `[A]` | **Страница** (статья) | `[A]Frontmatter{frontmatter}.svx` → URL: `/docs/settings/frontmatter` |

### Создание новой страницы

1. Создайте `.svx` файл внутри директории секции документации.
2. Используйте паттерн именования `[A]Заголовок{slug}.svx`.
3. Добавьте frontmatter в начале файла:

```md
---
title: Моя новая страница
description: Краткое описание для SEO и навигации.
---

# Моя новая страница

Здесь контент...
```

4. Чтобы объединить страницы в категории, создайте подкаталог `[C]Название категории{slug}/` и поместите `.svx` файлы внутри него.

5. Навигация, sitemap, поисковый индекс и оглавление генерируются автоматически — ручная конфигурация не требуется.

### Типы страниц

- **`.svx`** — Markdown со встроенными компонентами Svelte (для большинства контента)
- **`.svelte`** — Полноценный компонент Svelte (для интерактивных кастомных страниц)

## Полная документация

Шаблон поставляется с самодокументирующимися страницами: frontmatter, форматирование, брендинг, система имён, кастомные страницы и предпросмотр компонентов. Откройте `/docs` после запуска dev-сервера или прочитайте исходники в:

```
apps/web/src/lib/content/[N]Документация{docs}/
```

Документация исходного проекта: https://opensophy.com/docs/opensophy-docs

## Технологический стек

- **SvelteKit** — фреймворк, SSR/SSG, маршрутизация
- **MDsveX** — Markdown с компонентами Svelte
- **Tailwind CSS v4** — стилизация, тёмная тема через CSS-переменные
- **Shiki** — подсветка синтаксиса
- **KaTeX** — математические формулы
- **carbon-icons-svelte** — UI-иконки
